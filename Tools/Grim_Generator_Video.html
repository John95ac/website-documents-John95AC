­¤ō¢ How it works

  • Grim Size (Grid): Sets the maximum layout size for each sprite sheet image. For example, 8x8 will fit 64 frames in a single image before creating a new one.
  • Frames Per Second (FPS): Determines how many still frames to capture per second of video. Higher FPS means smoother animation but more output files.
  • Format & Compression: Adjusts the WebP image quality. Lower values reduce the final file size but may lower visual clarity.
  • Resolution Scale: Scales down the output frame dimensions by a factor. For example, a 1920x1080 video at 1/2 scale produces 960x540 frames. The Output resolution is shown next to the original in the video info bar. Snapshots are also generated at the selected scale.
  • Audio Format: Choose between WAV (lossless, larger) or MP3 (compressed, smaller). The audio file is named after the video. Check "Include in ZIP" to add it to the sprite sheet archive, or use "Generate Audio" to extract only the audio in a ZIP.
  • Audio Mixer: Upload multiple audio tracks. The first track becomes the Master (duration reference). Other tracks can be dragged along the timeline via the offset bar. Adjust volume per track (0-100%). Mix downloads as a single audio file inside a ZIP. Choose WAV or MP3 format for the mix output.
  • Voice to Text: Uses your browser's Speech Recognition API to transcribe spoken audio in real time. Select your recognition language (saved automatically). Start recording to capture audio and transcribe. Use "Export Audio" to download your recording as WAV or MP3 inside a ZIP.
  • Generate ZIP: All processing is done locally in your browser. The final ZIP contains all the generated sprite sheets, the extracted audio, and a config.ini file.
