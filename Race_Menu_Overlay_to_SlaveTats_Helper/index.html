A comprehensive Python tool for Skyrim modders

Designed to streamline the creation, conversion, and management of character overlay mods between the RaceMenu and SlaveTats systems. Built with PyQt5, it features a modern dark theme interface for ease of use.

Installation

Program Download

The program is available in two versions: one in Python and the other as an EXE. Choose the one you prefer. The EXE version is a port of the original Python script and is about 300 MB, while the Python version is only 100 MB. There is no functional difference between them. Simply download the program and place it wherever you like just remember the location. The EXE has an icon; click it to start the program. It is normal for it to take a moment to open.

SSEEDIT Script Installation

Simply install it like any other mod. Once installed, the new Pascal script will appear in SSEEdit. This will be used in one of the final steps to create the port.

SlaveTats - RaceMenu steps

First step - automatic psc papyrus creation

Using the Program to Generate the PSC File

Start the program and keep the first tab, 'SlaveTats - RaceMenu', open. Navigate to the directory of the tattoo mod you wish to convert. Inside that mod's folder, locate the following path: (Mod name)\textures\actors\character\slavetats You will find a JSON file in this directory. Drag and drop this file into the program window. A preview of the Papyrus (PSC) script will be automatically displayed. Click the Generate .psc File button to generate the script file.

Organizing Textures and Files for the Next Step

Once the folder containing the newly created PSC file opens, you can close it. Proceed by clicking the create organized texture folder button. This action will copy the mod's textures into a new, organized folder structure required for the new mod. Finally, move the two newly created folders into your mod's main directory, as demonstrated in the accompanying video. This completes the first step. The next stage is to convert the PSC file into a PEX file.

Second Step - Easily Compile PSC to PEX

Now that your PSC file is ready, the next step is to compile it into PEX format. While there are several methods to do this, such as using the official Creation Kit, direct command-line compilation, or a setup in Visual Studio Code, the method I use most frequently for its speed and efficiency is the PCA SE compiler. To make this even simpler, I've created a guided process within the tool.

Guided Compilation Steps

As shown in the video, navigate to the (Mod name)\scripts\Source directory to find your PSC file. In our conversion tool, go to the PCA Papyrus Compiler Helper tab.

  1. Click the gear icon and select "Emulate MO2" to generate a path, which you must copy.
  2. Open the PCA program and paste the copied path into its MO2 configuration section.
  3. Return to our tool, press the green button, and drag your PSC file into the window.
  4. Finally, drag the same PSC file into the PCA program to compile it.

Upon completion, the compiled PEX file will be automatically placed in the correct directory.

Third Step - Create the ESP/ESPFE with SSEEdit

Launching SSEEdit for the Final Step

The final piece is the plugin file (.esp). While this is typically done via the Creation Kit, I've developed a streamlined process using SSEEdit.

  1. Open SSEEdit through your mod manager (e.g., MO2). When prompted, select only the Example for creating racemenu overlays John95.esp file and wait for it to load.
  2. As shown in the video, right-click the template mod, choose "Apply Script," and select 0002 Race Menu Overlays ESP Creator.pas.
  3. A dialog will appear. You can create a new ESP, or an ESP flagged as an ESPFE (recommended).You will be prompted for a mod name and then the PEX file name. Enter your desired names.
  4. Navigate into your new plugin to the "Quest" section and manually edit the "FormID" field to match the new quest's ID, as shown in the video.
  5. Finally, run "Clean Masters" on your plugin, then close SSEEdit and save.

Cleanup and Organization (Optional) - This final step is recommended to save space and keep your mod tidy

  • Move the new ESP file you just saved into your main mod folder.
  • Since the mod is now a RaceMenu overlay, you can delete the now-unused SlaveTats texture folder, located at (Mod name)\t extures\actors\character\slavetats.

That's it! For even greater space savings, I will be posting a guide on using assisted compression techniques, which can shrink a 4GB mod down to 60MB without quality loss.

Process Explanation

1. JSON Analysis

The tool parses the JSON file and extracts the name, section, texture, and area from each entry.

2. PSC Generation

Creates the Papyrus script with appropriate AddBodyPaint/AddWarpaint calls depending on the area.

3. Overlays Structure

Organizes the DDS textures into the structure required by RaceMenu in Overlays/<section>/

4. Compilation

Uses the PCA Helper to automatically compile PSC to PEX with MO2 or standalone mode.

5. ESP Plugin

Uses the Pascal script in SSEdit to create the ESP plugin with the assigned quest.
